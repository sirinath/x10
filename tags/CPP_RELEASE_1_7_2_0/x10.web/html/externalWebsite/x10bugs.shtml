Reporting a bug in X10


    
 
Before reporting a bug

Before reporting a bug, please:

  • Search the bug database first to see if the problem has already been reported.
  • Ensure you have the latest version of X10.

Submitting a bug report

Please follow these guidelines when submitting a bug report:

  • Include the version numbers of affected components e.g. X10, Java and X10DT
  • Describe as best you can how to reproduce the problem. If possible, include a small X10 program which triggers the problem, and include or describe both the expected result and what actually happened.
  • Include any traceback, diagnostic or error messages produced.
  • Report only one problem for each bug report.

 
Feedback

For general feedback, contact the X10 team.
