Implicit syntax
§ Use conventional syntax for
operations on values of
remote type:
§  x.f = e //write x.f of type T
      final T v = e;
          finish async(x.loc) {
             x.f=v;
          }
§ … = …x.f …//read x.f of type T
     
future<T>(x.loc){x.f}.force()
§ Similarly for array reads and
writes.
§ Invoke a method
synchronously on values of
remote type
§ e.m(e1,…,en);
    
     final T  v  = e;
     final T1 v1 = e1;
     …
     final Tn vn = en;
    finish async (v.loc) {
      v.m(v1,…,vn);
    }
§ Similarly for methods
returning values.
